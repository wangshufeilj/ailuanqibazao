# 公考资料分析数据源

本目录用于集中存放**适合作为公务员/事业单位考试「资料分析」题目训练素材**的统计公报、行业年鉴、政府经济数据等一手数据源。

## 收录原则

1. 原则上为**统计公报、年度报告、行业数据**等含**结构化数字、同比/环比、比重、增速**的权威材料，便于出题与计算训练。
2. 以**一手来源**为主（政府机关、统计局、行业主管部门、央企年报等）。
3. 每个条目**不改变原分类归档**：文件原本仍按 `file-classification.mdc` 分在 `chinese-documents/`、`reference/` 等目录；此处**仅复制一份副本**作为"数据源专题"入口。
4. 标签约定：原文件 frontmatter 的 `tags` 中必须含有 **`公考资料分析数据源`**，便于反向检索。

## 触发流程（Agent 自动执行）

当用户明确表述「这是公考资料分析数据源」「打上公考资料分析数据源」「把它加入数据源」或类似意图时，Agent 按以下顺序执行：

1. 在**原文件**（含 PDF 配套的 `.md` 元信息文件、或 Markdown 精读笔记文件）frontmatter 的 `tags` 中追加 `公考资料分析数据源`（若已存在则不重复）。
2. 把下列文件**复制一份到本目录**（保持同名，不改序号、不改命名）：
   - 文件本体（PDF / XLSX 等原始数据文件）；
   - 对应的 `.md` 元信息或精读笔记（若有）。
3. 向用户汇报：原文件路径、此处副本路径、已追加的标签。

## 容量约束（强制）

本目录**同样遵守** `file-classification.mdc` 的**30 篇阈值强制细分规则**：

- 每次向本目录复制新副本前，先统计目录内现有 `.md`（不计 `README.md`）数量；
- 若 ≥ 30，**立即停手**，向用户上报并由用户决定细分维度（年度、地域、发布机构、领域等）后再落库；
- 子目录递归适用同一阈值；仓库总深度仍不超过 4 层。

详见 [.cursor/rules/civil-service-data-source.mdc](../.cursor/rules/civil-service-data-source.mdc) 第六节。
